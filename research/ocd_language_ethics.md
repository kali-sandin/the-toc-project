# Lenguaje responsable: TOC clínico vs. incomodidad visual

Este proyecto usa “TOC visual” como shorthand interno para una sensación cotidiana de desajuste, pero conviene separar esa broma del trastorno obsesivo-compulsivo real. Si la web va a informar de verdad, debe decirlo sin sermón y sin matar el concepto.

## Qué afirma la fuente clínica

La International OCD Foundation recuerda que el TOC no es una preferencia estética ni una manía simpática. Es un trastorno serio que aparece cuando una persona queda atrapada en un ciclo de **obsesiones** y **compulsiones**.

- **Obsesiones:** pensamientos, imágenes o impulsos intrusivos, no deseados, repetidos y angustiosos.
- **Compulsiones:** conductas o actos mentales usados para intentar reducir la angustia o neutralizar la obsesión.
- Para diagnóstico, el ciclo consume mucho tiempo, causa angustia intensa o interfiere con actividades importantes.

Fuente: International OCD Foundation, “About OCD”: https://iocdf.org/about-ocd/

## Cómo lo traducimos al producto

La web puede provocar una incomodidad visual controlada, pero no debe vender eso como “tener TOC”. Lo correcto es explicar que estamos trabajando con:

1. **expectativas perceptivas**: alineación, ritmo, simetría y agrupación;
2. **carga cognitiva extrínseca**: esfuerzo adicional que no ayuda a cumplir una tarea;
3. **fricción estética**: señales visuales que reducen confianza o comodidad;
4. **lenguaje responsable**: el nombre del proyecto es provocador, pero la página debe aclarar la diferencia.

## Cómo se invierte en UI sin banalizar

- Añadir una nota visible cerca de `#evidence`: “Esto no describe el TOC clínico; describe incomodidad visual inducida”.
- Mantener el tono incómodo, pero usar la nota para convertir la provocación en educación.
- Enlazar una fuente clínica externa y una pieza interna de research.
- Evitar frases tipo “todos somos un poco TOC”. La IOCDF lo señala explícitamente como misconception.

## Guardarraíl editorial

Si una rareza visual solo funciona porque ridiculiza un diagnóstico, no sirve. Si funciona porque muestra cómo la percepción busca orden y cómo una interfaz puede traicionarlo, sí encaja con el proyecto.

## Oportunidades accionables

- Crear un bloque `nota clínica / nota visual` en `#evidence` con diseño torcido pero lectura clara.
- Añadir un test manual al checklist: la nota debe ser visible en modo TOC y en modo calma.
- Si el proyecto crece, valorar renombrar el claim principal de “TOC” a “picor visual” en copy explicativo, manteniendo el título como pieza provocadora.
